# Using Dijkstra's Algorithm to Solve Guitar Chord Progressions

When you play a chord progression on guitar, there are usually multiple ways to finger each chord. A C major chord alone has at least five common voicings spread across the neck. The challenge is picking voicings that flow smoothly from one chord to the next — minimizing the distance your hand has to travel.

This is fundamentally a shortest-path problem, and Dijkstra's algorithm is a natural fit.

## The Problem

Given a sequence of chord names like `Cmaj7 → Am7 → Dm7 → G7`, find the combination of voicings that minimizes total hand movement across the progression.

Each chord has multiple possible voicings (positions on the fretboard). The "cost" of transitioning between two voicings is based on how far your fingers need to move.

## Modeling It as a Graph

I model this as a weighted directed graph:

- **Nodes**: Each possible voicing of each chord in the progression
- **Edges**: Connections from every voicing of chord N to every voicing of chord N+1
- **Weights**: A distance metric based on finger movement between two voicings

The distance metric considers the average fret position difference and individual finger movements, weighted to penalize large jumps.

## The Result

Dijkstra's algorithm finds the shortest path through this graph — the sequence of voicings with minimal total transition cost. The result is a set of fingering charts that flow naturally from one to the next.
